You should always be running server side validation as well as client side validation. Client side validation on it's own is never sufficient. Malicious users can easily send things into your server if you do not do this.

Using built in form-validation:

The ability to validate user data without relying on javascript is one of the most significant features of HTML5 form controls. This is done by using attributes on form elements.

  • required: Specifies if the form needs to be filled out.
  • minlength: & maxlength: Specifies the minimum and maximum length of textual data(strings)
  • min & max: Specifies the minimum and maximum values of numerical input types.
  • type: Specifies whether the data needs to be a number, an email address or some other specific preset type.
  • pattern: Specifies a regular expression that defines a pattern the entered data needs to follow.

If the data entered follows the rules specified by the attributes then it is considered valid. If not, it is considered invalid. When they are valid you can style the element with the:valid CSS pseudo-class, which lets you apply a specific style to valid elements. You can submit the form, nothing is stopping it.

If the form is invalid then you can style the field a certain way using the :invalid CSS psuedo-class, and sometimes other UI psuedo-classes depending on the error, which allows you to apply a specific style to invalid. And if the user tries to second the data the browser will block the form and display an error message.

Regular Expression

Regex, here are some below to give a basic idea:

  • a -- Matches one character that is a (not b, not aa, and so on).
  • abs -- Matches a, followed by b, followed by c.
  • ab?c -- Matches a, optionally followed by a single b, followed by c. (ac or abc)
  • ab*c -- Matches a, optionally followed by any number of b's, followed by c. (ac, abc, abbbbbbbc and so on.)
  • a|b -- Matches one character that is a or b.
  • abc|xyz -- Matches exactly abc or exactly xyz (but not abcxyz or a or y, and so on.)

Validating Forms using Javascript

Using the Constraint Validation API consists of methods and properties on the following form element DOM interfaces:

  • HTMLButtonElement (represents a button element)
  • HTMLFieldSetElement (represents a fieldset element)
  • HTMLInputElement (represents an input element)
  • HTMLOutputElement (represents an output element)
  • HTMLSelectElement (represents an select element)
  • HTMLTextAreaElement (represents a textarea element)

The Constraint Validation API makes the following properties available on the above elements:

validationMessage: Returns a localized message describing the validation constraints that the control doesn't satisfy (if any). If it is not a candidate for constraint validation (willValidate is false) or the element value satisfies its constraints (is valid) this will return an empty string.

validity: Returns a ValidityState object that contains several properties describing the validity state of the element. You can find full list of them here.

willValidate: Returns true if the element will be validated when the form is submitted; false otherwise.
