Object Oriented Programming

What is object oriented programming? Object oriented programming is a programming paradigm centered around objects rather than functions. It is not a programming language or tool. An object is a portion of code that contains functions that are related in some way. Watch a video explaining this method of programming here.

Object oriented programming is a big deal to recruiting companies. It's something work learning and showing on a resume.

4 Pillars of Object Oriented Programming

  1. Encapsulation
  2. Abstraction
  3. Inheritance
  4. Polymorphism

In Object Oriented Programming the functions are referred to as methods, and the variables are referred to as properties.

Encapsulation

Encapsulation is when we group related properties and methods that operate on them into objects. Example: Check JS in console to find the different in the object oriented way vs. the way that isn't.

Abstraction

We can hide some of the methods and properties from the outside. It creates a simpler interface of an object. It also helps reduce the impact of change. None of the changes will leak to the outside of the object.

Inheritance

Eliminates redundant code. HTML elements like CheckBox, TextBox, and Select all have properties like hidden and innerHTML and methods like click() and focus() instead of redefining all of the HTML functions we can define these into one HTML object called HTMLElement and have other objects inherit these properties and methods.
